## 1. 🎯 프로젝트 개요

| 구분 | 내용 |
| :--- | :--- |
| 한 줄 소개 | 개인이 수입/지출을 기록하고 카테고리별·월별 통계를 확인하는 가계부 REST API 서비스 |
| 대상 사용자 | 로그인한 개인 사용자 (인증된 본인 데이터만 접근 가능) |
| 핵심 목표 | Spring Boot, JWT, JPA 기반 백엔드 아키텍처 및 RESTful API 구현과 Docker + EC2 자동 배포 완주 |

## 2. 👤 유저 스토리 (User Stories)

| # | 유저 스토리 (~로서 나는 ~하고 싶다, 왜냐하면 ~) |
| :---: | :--- |
| US-1 | 신규 사용자로서 나는 이메일과 비밀번호로 회원가입을 하고 싶다, 왜냐하면 나만의 개인 가계부 계정을 생성하고 싶기 때문이다 |
| US-2 | 기존 사용자로서 나는 로그인을 하여 JWT Access Token을 발급받고 싶다, 왜냐하면 인증이 필요한 가계부 기능에 보안 접근하고 싶기 때문이다. |
| US-3 | 가계부 사용자로서 나는 수입/지출 카테고리를 조회하고 커스텀 카테고리를 추가하고 싶다, 왜냐하면 자산 흐름을 정확히 기록하고 싶기 때문이다. |
| US-5 | 가계부 사용자로서 나는 월별 거래 내역을 페이징 목록으로 조회하고 싶다, 왜냐하면 지출 내역을 부담 없이 한눈에 모아보고 싶기 때문이다. |
| US-6 | 가계부 사용자로서 나는 월별 총수입·지출·잔액 및 카테고리별 지출 비중(%)을 조회하고 싶다, 왜냐하면 월별 소비 패턴과 지출 구조를 한눈에 파악하고 싶기 때문이다. |

## 3. ⚙️ 기능 요구사항 (MoSCoW)

| ID | 기능 | 상세 설명 | 우선순위 |
| :---: | :--- | :--- | :---: |
| F-01 | 회원가입/로그인 | BCrypt 비밀번호 암호화 저장, JWT 토큰 발급 및 로그인 검증 | Must |
| F-02 | 카테고리 관리 | 기본 카테고리 시드 데이터 제공 + 사용자 정의 카테고리 CRUD | Must |
| F-03 | 거래내역 CRUD | 수입/지출 내역 등록·상세조회·수정·삭제 | Must |
| F-04 | 목록 필터·페이징 | yearMonth, type, categoryId 조건 필터링 + page/size 페이징 응답 | Must |
| F-05 | 월별 통계 | 총수입·총지출·잔액 계산 및 카테고리별 지출 금액/비율(%) 집계 | Must |
| F-06 | 인가 (Security) | Spring Security + JWT 필터를 활용한 본인 소유 데이터만 접근 제어 (403 Forbidden) | Must |
| F-07 | 검증·예외 처리 | @Valid (금액>0, 날짜 필수) 및 공통 응답 규격 ({success, code, message, data}) 적용 | Must |
| F-08 | 프론트 4종 | 로그인/회원가입, 거래 목록, 거래 등록/수정, 월별 통계 화면 흐름 지원 | Must |
| F-09 | Swagger | Springdoc OpenAPI 기반 REST API 명세 자동화 문서 구축 | Should |
| F-10 | 배포 | Dockerizing + GitHub Actions CI/CD + AWS EC2 실배포 환경 구축 | Must |
| F-11 | (도전) 엑셀 내보내기 | 월별 거래 내역 데이터를 CSV/Excel 파일로 다운로드 기능 | Could |

## 4. 🛡️ 비기능 요구사항

| 분류 | 요구사항 | 구체적 기준 및 구현 방식 |
| :---: | :--- | :--- |
| 보안 | 본인 데이터만 접근 | 모든 DB 조회/수정/삭제 쿼리에 user_id 조건 필수 적용 (타인 자원 접근 시 403 Forbidden) |
| 보안 | 비밀번호 암호화 | PasswordEncoder (BCrypt) 알고리즘 적용 (평문 저장 절대 불가) |
| 성능 | 목록 페이징 및 최적화 | Spring Data JPA Pageable 기반 페이징 적용, 카테고리 매핑 시 findAllById 배치 조회로 N+1 방지 |
| 표준성 | 공통 응답/에러 포맷 | 모든 성공/실패 응답을 단일 JSON 구조 ({success, code, message, data})로 통일 |
| 배포 | 실접속 환경 제공 | AWS EC2 인스턴스에 Docker 컨테이너 기반으로 배포하여 외부 접속 가능한 퍼블릭 URL 제공 |

## 5. 📌 범위 (Scope)

### 1) 기본 과제 (In Scope)
- 이메일/비밀번호 기반 JWT 회원가입 및 로그인
- 카테고리 시드 데이터 제공 및 CRUD
- 수입/지출 거래 내역 CRUD 및 페이징 목록 조회
- 월별 총수입/총지출/잔액 및 카테고리별 비중 통계 API
- ApiResponse<T> 및 GlobalExceptionHandler 공통 아키텍처 구축
- Docker + GitHub Actions CI/CD + AWS EC2 배포

### 2) 도전 과제 (Stretch)
- Swagger (Springdoc) API 자동 문서화
- 거래 내역 검색 및 조건별 다중 필터링

### 3) 이번엔 안 함 (Out of Scope)
- OAuth2 소셜 로그인 (카카오, 구글 등)
- Refresh Token DB 저장 및 자동 재발급
- 실시간 은행/카드사 데이터 파싱 연동
